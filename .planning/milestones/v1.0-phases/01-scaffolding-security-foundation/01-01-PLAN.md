} from "@copilotkit/runtime";

};

<threat_model>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser ↔ Next.js server | All client code runs in untrusted user environment; secrets must never cross this boundary |
| Repo ↔ Public git history | Once committed, secrets cannot be unlearned even if force-pushed (caches, mirrors, GitHub PR audit logs persist) |
| Application server ↔ Anthropic API | `ANTHROPIC_API_KEY` authorizes calls; leakage = financial / abuse risk on the user's Anthropic account |
| Application server ↔ GitHub API | `GITHUB_TOKEN` authorizes PR diff reads; classic PAT with `public_repo` scope minimizes blast radius (Phase 2 concern, but env wiring established now) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | `process.env.ANTHROPIC_API_KEY` accessed in a Client Component due to a `NEXT_PUBLIC_ANTHROPIC_API_KEY` typo | mitigate | (a) Task 1 grep gate fails the verify if any `NEXT_PUBLIC_ANTHROPIC` or `NEXT_PUBLIC_GITHUB` string exists in repo; (b) Task 1 `.env.local.example` template explicitly documents the prohibition with inline comments; (c) Task 2 verify forbids `NEXT_PUBLIC_` in `app/api/copilotkit/route.ts`; (d) `lib/server-only.ts` is created so Phase 2 modules reading the env var get a build-time error if accidentally bundled client-side |
| T-01-02 | Information Disclosure | `.env.local` containing real secrets is committed to git history | mitigate | Task 1 appends explicit rules to `.gitignore` covering `.env`, `.env.local`, `.env*.local`, `.env.development.local`, `.env.production.local` with an explicit allow-list (`!.env.local.example`) so the template is committable but real secrets cannot be. Task 3 verifies via `git check-ignore -v .env.local` |
| T-01-03 | Information Disclosure | `BuiltInAgent` or `AnthropicAdapter` instantiated client-side, leaking the SDK + key | mitigate | Route handler lives at `app/api/copilotkit/route.ts` (Route Handler — server-only by Next.js convention). Task 2 verify forbids `NEXT_PUBLIC_` in the route file. The `@copilotkit/runtime` package itself imports server-only modules; Next.js will throw a build-time error if it ends up in a client bundle |
| T-01-04 | Information Disclosure | A new developer onboards, doesn't see env var requirements, and either commits a real `.env.local` OR adds `NEXT_PUBLIC_` to make it "work" in a Client Component | mitigate | `.env.local.example` is committed at repo root with explicit comments for each var: "NEVER prefix with NEXT_PUBLIC_ — this would bake the token into the JS bundle". The comment is the developer-facing security warning |
| T-01-05 | Tampering | CopilotKit `runtimeUrl` points to a third-party host instead of `/api/copilotkit`, exfiltrating user requests + ultimately the API key via the malicious endpoint's response | mitigate | Task 2 verify hard-codes the check `runtimeUrl="/api/copilotkit"` in `app/providers.tsx` (relative URL, same-origin only).
